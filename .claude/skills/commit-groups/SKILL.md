## Current state
- Status: !`git status --short`
- Diff stat: !`git diff --stat`

Group all modified/untracked files into logical commits. Each commit should be a single coherent change (e.g. one bug fix, one feature, one refactor — not a mix).

## Rules (non-negotiable)
- Commit message format: `type: description` where type is one of `feat`, `fix`, `refactor`, `build`, `test`, `chore`, `docs`
- Message must be 15 words or fewer (count the words, not the characters)
- No co-author lines, no "Generated by", no Claude attribution of any kind
- Stage only the files for the current group before committing — never `git add -A`
- Never use `--no-verify`
- Commit messages must be passed via heredoc to avoid shell escaping issues

## Process
1. Run `git diff` to read all changes in full
2. Mentally group files by what problem they solve together
3. For each group (smallest logical unit first, most foundational first):
   a. `git add <specific files>`
   b. `git commit -m "$(cat <<'EOF'\ntype: description\nEOF\n)"`
4. After all commits, run `git log --oneline -10` to show what was committed
